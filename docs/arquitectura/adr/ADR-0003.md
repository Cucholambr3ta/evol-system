# ADR-0003: Setup repo autonomo + pentest nativo sin Shannon

**Estado:** Aceptado
**Fecha:** 2026-06-04

## Contexto

Evol-DD necesita paridad con el sistema de referencia: setup inicial del repo (existente /
crear en nube / local) + modo dev/collab, y un arsenal de seguridad/pentest. Evol-DD NO
usa Shannon (AGPL) — X-DD lo usa solo opt-in.

## Decision

1. `/evol setup-repo` (paso 0): pregunta ubicacion + modo. `evol-gitflow.sh setup --create`
   crea repo en GitHub autonomamente (gh-gated, private default), `--local` para solo local.

2. Arsenal de seguridad 100% nativo + externas permisivas (auto-discovery), SIN Shannon:
   - Nativas (Python stdlib): evol-scan, evol-shield, evol-crash, evol-patch, evol-validate,
     STRIDE modeling, fuzz estatico.
   - Externas auto-discovery: semgrep, gitleaks, trivy, nuclei, OWASP ZAP.
   - Las 2 capacidades que X-DD delega a Shannon (dynamic fuzz URL viva + exploit verify):
     Evol-DD usa nuclei/ZAP para dynamic; exploit-verify queda como tarea MANUAL en el
     checklist (sin auto-verify AGPL). Cero dependencia AGPL.

## Consecuencias

- Evol-DD MIT no se contamina (no bundlea ni consume Shannon).
- Cobertura de pentest completa con herramientas permisivas + manual para exploit-verify.
- `evol-security-inventory.py` no referencia Shannon.
